# مستندات BaleTunnel

## معرفی

BaleTunnel ابزاری برای عبور از محدودیت‌های اینترنتی است که با بهره‌گیری از زیرساخت پیام‌رسان بله و پروتکل WebRTC، یک تانل رمزنگاری‌شده بین کلاینت و سرور ایجاد می‌کند. ترافیک شبکه به صورت یک تماس صوتی عادی در بستر بله ظاهر شده و از طریق دیتاچنل LiveKit منتقل می‌شود.

## معماری

```
[کلاینت] --> SOCKS5 --> [LiveKit DataChannel] --> [سرور] --> اینترنت
```

- کلاینت یک پروکسی SOCKS5 محلی ایجاد می‌کند
- ترافیک از طریق WebRTC DataChannel به سرور ارسال می‌شود
- سرور ترافیک را به مقصد نهایی هدایت می‌کند
- تمام ارتباطات رمزنگاری‌شده هستند

## نصب

### پیش‌نیازها

- Python 3.10 یا بالاتر
- اتصال اینترنت (حداقل برای دسترسی به سرورهای بله)
- حساب کاربری فعال در پیام‌رسان بله

### مراحل نصب

```bash
cd BaleTunnel
pip install -r requirements.txt
```

## راه‌اندازی سرور

سرور باید روی ماشینی با اینترنت بدون محدودیت اجرا شود.

```bash
python src/app.py server
```

### ورود به حساب

```
> login
Phone number (e.g. +98912...): +989121234567
Code sent. Registered: True
Enter verification code: 12345
Login successful.
```

### مدیریت درخواست‌ها

هنگامی که کلاینتی درخواست اتصال ارسال می‌کند، در لیست انتظار قرار می‌گیرد:

```
> pending
Pending calls (1):
  Call 5913263170600257563 from caller 912116268
```

قبول درخواست:
```
> accept 5913263170600257563
```

قبول و ذخیره در لیست مجاز (اتصال‌های بعدی خودکار پذیرفته می‌شوند):
```
> accept 5913263170600257563 --save
```

رد درخواست و مسدودسازی:
```
> reject 5913263170600257563 --block
```

### مدیریت لیست دسترسی

```
> admission              نمایش لیست مجاز
> allow 912116268        افزودن دستی به لیست مجاز
> block 912116268        مسدود کردن
> unblock 912116268      رفع مسدودیت
> blacklist              نمایش لیست مسدود
```

### مشاهده و مدیریت کلاینت‌ها

```
> clients

  [5913263170600257563] caller 912116268 rx=1024B tx=2048B

> kick 5913263170600257563
Kick: ok
```

### قطع تمام اتصالات


## راه‌اندازی کلاینت

کلاینت روی دستگاه کاربر اجرا شده و یک پروکسی SOCKS5 محلی ایجاد می‌کند.

```bash
python src/app.py client --port 1080
```

### مراحل اتصال

1. ورود به حساب:
```
> login
Phone number (e.g. +98912...): +989129876543
Login successful.
```

2. مشاهده لیست مخاطبین:
```
> peers
Contacts (2):
  [1613707444] ermia (@ermiah72)
  [912116268] user2
```

3. اتصال به سرور:
```
> connect 1613707444
Connecting to peer 1613707444...
[Tunnel/C] WebRTC tunnel ready
```

### تنظیم پروکسی

پس از نمایش پیام "WebRTC tunnel ready"، پروکسی SOCKS5 فعال است.

خط فرمان:
```bash
curl --socks5-hostname 127.0.0.1:1080 http://ifconfig.me
```

مرورگر Firefox:
- Settings > Network Settings > Manual proxy configuration
- SOCKS Host: 127.0.0.1
- Port: 1080
- SOCKS v5
- گزینه "Proxy DNS when using SOCKS v5" را فعال کنید

تنظیمات سیستم:
- پروکسی SOCKS5 را روی آدرس 127.0.0.1 و پورت 1080 تنظیم کنید

## عیب‌یابی

| خطا | علت و راه‌حل |
|------|-------------|
| InvalidPeer | شناسه سرور اشتباه است. از دستور peers برای یافتن شناسه صحیح استفاده کنید |
| Timeout | اتصال اینترنت ناپایدار است یا سرور آفلاین است |
| Session expired | نشست منقضی شده. logout و سپس login مجدد انجام دهید |
| SSL error | فایل ca_bundle.pem باید در پوشه src قرار داشته باشد |
| WS Reconnecting | طبیعی است. اتصال WebSocket به صورت خودکار بازیابی می‌شود |

## نکات امنیتی

- فایل پیکربندی حاوی توکن احراز هویت است. از اشتراک‌گذاری آن خودداری کنید.
- پروکسی به صورت پیش‌فرض روی تمام اینترفیس‌ها (0.0.0.0) گوش می‌دهد. در صورت نیاز به محدودسازی، فایروال تنظیم کنید.
- DNS حتما باید از طریق پروکسی ارسال شود (socks5-hostname نه socks5).
